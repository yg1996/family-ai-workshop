# ✅ PREP — E2E checklist to session day

Session: **~10 days out**. Work top to bottom — P0 first, everything else is polish on top of working demos.
Rule of thumb: **if a demo fails live, the talk fails; if the talk stumbles, nobody cares — so demos first, rehearsal second, logistics last.**

---

## 🔴 P0 — Demos must work (days 1–3, ~4h total)

Everything below runs on the **demo account** (yours). Full scripts: [demo-script.md](demo-script.md).

### Demo account baseline (~20 min)
- [ ] Plus active; app + web logged in
- [ ] Custom instructions **emptied** (Demo 1 fills them live) — save your current ones somewhere first
- [ ] Memory: review Settings → Personalization → Memory; delete anything weird/private that could pop up on the TV
- [ ] Verify visible in UI: Deep Research button, Projects in sidebar, Tasks (ask "show my scheduled tasks")

### Demo 0 — deep research dry-run (~40 min, mostly waiting)
- [ ] Run the exact Demo 0 prompt once, **time it** — need: finishes in 10–40 min (must be done by minute ~55 of the session)
- [ ] Read the output — impressive? If thin, sharpen the prompt (more comparison criteria = richer report)
- [ ] Note actual duration in demo-script.md margin

### Demo 1 — custom instructions before/after (~20 min)
- [ ] Run "before" (empty instructions) — confirm generic answer
- [ ] Paste prepared instructions, run "after" in NEW chat — confirm dramatic contrast (Hebrew, kosher, shopping list)
- [ ] Delete instructions again after test (must be empty at session start!)

### Demo 2 + 2b — PDF + hallucination (~30 min)
- [ ] Pick THE PDF: electricity bill / arnona / appliance manual — something parents recognize; save to demo device
- [ ] Upload + run both questions — answers grounded and clear?
- [ ] **Test hallucination example** (clause 7.3 rental-law quote) — did it confidently invent? If it hedged/refused: test fallback (obscure sports season standings / local product prices) until you have one that reliably errs
- [ ] Write down which example worked — that's the one you use

### Demo 3 — Project + Task (~20 min)
- [ ] Create a throwaway Project, paste instructions, upload file, ask the question — smooth?
- [ ] Create the weekly flight-price Task — confirm it registers; then delete it (recreate live at session)
- [ ] Delete throwaway Project (create live at session)

### Demo 4 — GPT Store picks (~30 min)
- [ ] ⚠️ **Verify all 4 GPTs still exist** (community GPTs vanish): TRAVEL Trip & Flight Itineraries · Construction Estimating Pro · image generator · SAP Expert
- [ ] Run each starter prompt (from [recipe card 8](recipe-cards/8-gpt-store-starter.md)) once — output decent?
- [ ] Any dead/bad one → pick replacement in store (good ratings, many users), update card 8 + deck slide 9
- [ ] Confirm TRAVEL GPT works as the live demo; fallback = image generator ("birthday card for grandma")

### Screen + room (~20 min)
- [ ] TV mirroring from demo device — test with the actual deck AND with ChatGPT open
- [ ] Font size readable from couch distance? (ChatGPT web: Cmd+ to zoom)

## 🟡 P1 — Know the material cold (days 4–7, ~5h total)

### Do the stuff you'll teach (~2.5h) — "see how to actually do it"
- [ ] Build [card 1 — Vacation Planner](recipe-cards/1-vacation-planner.md) end-to-end on your own account
- [ ] Build [card 7 — Morning Digest](recipe-cards/7-morning-digest-task.md) — let it actually fire next morning, see what arrives
- [ ] Build [card 8 — GPT Store](recipe-cards/8-gpt-store-starter.md) — install the picks yourself
- [ ] Skim-build cards 2–6: create each Project, paste instructions, one message each — enough to help anyone stuck
- [ ] Drill feature locations until instant: Settings → Personalization → Custom Instructions / Memory · sidebar → New Project · Explore GPTs · Deep Research button · Tasks list

### Rehearse the talk (~2h)
- [ ] Read [TALK-TRACK.md](TALK-TRACK.md) against the live deck once — adjust any line that doesn't feel like YOUR words
- [ ] Full timed run-through #1, alone, with real demos — target ≤65 min for the talk block
- [ ] Full run-through #2 for spouse/friend — collect "didn't get it" moments, fix
- [ ] Know your recovery lines: demo fails → "this is why we test in production" → fallback video / move on

### Family readiness (~30 min, phone calls)
- [ ] Audit per person: Plus? logged in? app updated? phone or laptop for hands-on?
- [ ] Anyone without Plus → decide: buy / share screen with a sibling / steer to card 8 (works fine on free-ish tiers)
- [ ] Tell everyone: bring charged device + charger

## 🟢 P2 — Logistics + backups (days 8–9, ~2h total)

- [ ] Print recipe cards (1 set per person) + cheat sheet — or invite family to the GitHub repo (private → needs invites; printing simpler for parents)
- [ ] During final rehearsal: **screen-record every demo** — that's your fallback if wifi/ChatGPT is down
- [ ] Export deck to PDF from Gamma (File → Export) — second backup on the demo device
- [ ] Devices charged, chargers staged, coffee/snacks plan

### Day before — dress rehearsal
- [ ] Full end-to-end with timer: slides + demos + one recipe build
- [ ] Demo account back to clean state (instructions empty, no leftover Project/Task from rehearsal)

---

## 📋 Day-of runsheet (30 min before)

1. TV mirroring on, deck open at slide 1
2. ChatGPT open in second window: instructions EMPTY, memory clean, PDF file on desktop
3. Deep Research prompt staged in a draft note — paste ready
4. Recipe cards + cheat sheets on the table
5. Fallback videos + PDF deck reachable in one click
6. Phone on silent. Breathe. You know this cold — you built all of it.
