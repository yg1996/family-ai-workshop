## 🇬🇧 English

**What it is:** A Project that turns screenshots and rough notes into polished user guides, rollout emails, and FAQ documents — in your voice, in your format.

**Why it's useful daily:** System implementation means explaining the same thing to many people, many times. Screenshot the screen, paste bullet notes, get a guide users can actually follow. *(Confidentiality still applies: mask customer data in screenshots.)*

### Build it — 5 steps

1. Sidebar → **New Project** → name it **"Guides & Rollouts"**
2. Project **Instructions** → paste the block below
3. Take 3–5 screenshots of a flow you're rolling out; upload with the **first message** below
4. Iterate: *"make step 4 clearer"*, *"add a troubleshooting section"*, *"now write the announcement email"*
5. Each new system/rollout = a new chat in the same project — format stays consistent

### 📋 Project instructions — copy, paste, edit

```text
I implement information systems and write training material for non-technical
users. Always answer in Hebrew unless I ask otherwise. When I upload screenshots,
turn them into step-by-step user guides: numbered steps, one action per step,
reference the screenshots ("as shown in image 2"), plain language, no jargon.
Every guide ends with a short FAQ and a "common mistakes" section. When I ask
for rollout emails: short, friendly, with what-changes-for-you first.
```

### 💬 First message (attach screenshots)

```text
I'm attaching screenshots of [flow name, e.g. "submitting a request in the new
system"]. Write a user guide for employees who have never seen this system.
Audience: [describe them]. Then suggest what screenshot I'm missing.
```

**✨ Make it yours:** Paste one guide you've already written and liked — tell it *"match this style from now on."*

## 🇮🇱 עברית

**מה זה:** פרויקט שהופך צילומי מסך והערות גולמיות למדריכי משתמש מלוטשים, מיילי הטמעה ומסמכי שאלות-תשובות — בקול שלכם, בפורמט שלכם.

**למה זה שימושי:** הטמעת מערכות זה להסביר את אותו דבר להרבה אנשים, הרבה פעמים. מצלמים את המסך, מדביקים נקודות, מקבלים מדריך שמשתמשים באמת יכולים לעקוב אחריו. *(כללי הסודיות עדיין חלים: להסתיר נתוני לקוחות בצילומי מסך.)*

### איך בונים — 5 שלבים

1. תפריט צד ← **New Project** ← קראו לו **"מדריכים והטמעות"**
2. **Instructions** של הפרויקט ← הדביקו את הבלוק למטה
3. צלמו 3–5 צילומי מסך של תהליך שאתם מטמיעים; העלו אותם עם **ההודעה הראשונה** למטה
4. שפרו בשיחה: *"תבהיר את שלב 4"*, *"תוסיף פרק פתרון תקלות"*, *"עכשיו כתוב את מייל ההכרזה"*
5. לכל מערכת/הטמעה חדשה — שיחה חדשה באותו פרויקט. הפורמט נשאר אחיד

### 📋 הוראות הפרויקט — העתק, הדבק, ערוך

</div>

```text
אני מטמיעת מערכות מידע וכותבת חומרי הדרכה למשתמשים לא-טכניים. תמיד ענה בעברית
אלא אם ביקשתי אחרת. כשאני מעלה צילומי מסך, הפוך אותם למדריך צעד-אחר-צעד: שלבים
ממוספרים, פעולה אחת בכל שלב, הפניה לצילומים ("כפי שמוצג בתמונה 2"), שפה פשוטה,
בלי ז'רגון. כל מדריך מסתיים בשאלות-תשובות קצרות ובפרק "טעויות נפוצות". כשאני
מבקשת מייל הטמעה: קצר, ידידותי, ומתחיל במה-משתנה-עבורך.
```
